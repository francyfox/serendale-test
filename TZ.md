# **Тестовое задание — Frontend Developer (Next.js)**

## **Общая задача**

Сверстать лендинг на основе макета:

👉 https://www.figma.com/design/R3Utyz8dCq6lt22lrZIBHF/Serendale.ai—AI-based-Blockchain-Hero-Exploration–Community-?node-id=0-1&m=dev&t=9Ir4TV0f07P262xO-1

Необходимо:

- Воспроизвести дизайн страницы в **десктопной версии**
- Реализовать **адаптацию под мобильные устройства**
- Добавить дополнительные секции:
    - Преимущества
    - Отзывы
    - Подвал сайта (footer)

Тематику сайта можно заменить (например: финансы, образование, маркетинг, AI-сервисы), но **визуальный стиль должен остаться близким к оригинальному макету**.

## **Технические требования**

### **Frontend стек**

- Next.js
- Tailwind CSS
- HTML5 семантическая разметка (header, main, section, footer, …)

### **Адаптивность**

Сайт должен корректно отображаться на:

- Desktop (1440px+)
- Tablet
- Mobile (320px и выше)

Использовать:

- Flexbox / Grid

### **Оптимизация**

- Скорость загрузки страницы (Lighthouse ≥ 90 желательно)
- Оптимизация изображений (предварительное сжатие в webp)

## **Дополнительные секции**

### **1. Блок преимуществ**

Добавить секцию с 3–6 преимуществами продукта.

Пример структуры:

- Иконка
- Заголовок
- Краткое описание

### **2. Блок отзывов**

Реализовать карточки отзывов пользователей.

Дополнительно:

- Добавить **JSON-LD микроразметку schema.org** для отзывов.

### **3. Footer**

В футере разместить:

- Навигационные ссылки
- Контакты

## **SEO и семантика**

- Использовать семантические теги
- Добавить:
    - Meta description
    - Title
    - OpenGraph (по возможности)

## **Что будет оцениваться**

### **Верстка**

- Соответствие макету
- Аккуратность Tailwind классов
- Чистота структуры проекта

### **Код**

- Читаемость
- Разделение компонентов
- Отсутствие дублирования

### **Производительность**

- Оптимизация изображений
- Lighthouse Performance

### **Дополнительно (будет плюсом)**

- Анимации (Framer Motion или CSS)
- Zustand / state management
- Базовая интерактивность
